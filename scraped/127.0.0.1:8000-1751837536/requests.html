Requests

This content has been extracted from the Requests website.

Most existing Python modules for sending HTTP requests are extremely verbose and cumbersome. Things shouldnt be this way. Not in Python.

While Pythons standard urllib2 module provides most of the HTTP capabilities you need, the API is thoroughly broken. It was built for a different time  and a different web. It requires an enormous amount of work (even method overrides) to perform the simplest of tasks.


Luckily, Requests is an Apache2 Licensed HTTP library, written in Python, for human beings.

Requests takes all of the work out of Python HTTP/1.1  making your integration with web services seamless. Theres no need to manually add query strings to your URLs, or to form-encode your POST data. Keep-alive and HTTP connection pooling are 100% automatic, powered by urllib3, which is embedded within Requests.


Using the Requests Module

Since we are using a module, the first thing to do is import it:

import requests


Making a Request

Making a request is quite easy. Let's request the ECTEG website:

r = requests.get( "http://www.ecteg.eu" )

This creates a Response object called r.The Response object contains a lot of useful information. For example:

  • Response URL Information
  • Response Content
  • Response Encoding Information
  • Response Status Codes
  • Response Headers

Requests simple API means that all forms of HTTP request are as obvious. Other requests such as POST, PUT, DELETE, HEAD and OPTIONS are all done in a similar manner to GET.

Response URL Information

This can be useful if you are managing a lot of response objects. The following command will print the URL of the request.

print ( r.url )


Response Content

This is one of the most useful items retrieved by a request. It is the actual content of a web page at a given URL. The following command can be used to print a web page:

print ( r.text )

Requests will automatically decode content from the server.


Response Encoding Information

The text encoding guessed by Requests is used when you access the response content. To get the encoding used, or to set your own encoding, use the following commands:

To get the encoding: print ( r.encoding )

To set the encoding: r.encoding = 'utf-8'


Response Status Codes

You can also check a response's status code to determine if a request succeeded, if there was content returned, if redirection was needed, etc.

A full list of status codes are described here.

The following command can be used to view the status code of a response:

print ( r.status_code )

Requests also comes with a built-in status code lookup object for easy reference. For example to determine if a request was successful (status code = 200), the following will print True or False:

print ( r.status_code == requests.codes.ok )

Response Headers

A server's response headers are returned as a case insensitive Python dictionary. The following command can be used to view this dictionary:

print ( r.headers )

This will return something like the following:


{
	'content-length': '2018', 
	'content-encoding': 'gzip', 
	'accept-ranges': 'bytes', 
	'vary': 'Accept-Encoding', 
	'server': 'Apache/2.4.7 (Ubuntu)', 
	'last-modified': 'Thu, 06 Aug 2015 16:16:10 GMT', 
	'etag': '"158b-51ca6d65d2909-gzip"', 
	'date': 'Fri, 07 Aug 2015 11:16:00 GMT', 
	'content-type': 'text/html'
}

These "key:values" combinations can be accessed like a normal dictionary. For example, the following will print the server information (note that case doesn't matter):

print ( r.headers['SERVER'] )
